## Design Context

### Users

Nidus serves Rust backend engineers and framework evaluators who want NestJS-like application organization without leaving Rust-native explicitness. They are likely reading the site while evaluating installability, API shape, examples, production defaults, and whether the framework is stable enough for a real service.

### Brand Personality

Sharp, nested, deliberate. The interface should feel technically rigorous and public-release ready, with a dark violet identity derived from the logo rather than a generic developer-tool theme.

### Aesthetic Direction

Use a near-black and dark violet structure with purple identity accents, sharp organic nesting forms inspired by the Nidus logo, and documentation-first information architecture. The site may take structural inspiration from NestJS's clear framework pitch and docs navigation, but it must not copy NestJS text or visual design. Avoid green backgrounds, generic AI gradients, glassy glow-heavy surfaces, and decorative visuals that do not explain the product.

### Design Principles

- Make installation, docs navigation, and real examples immediately visible.
- Keep the core facade and official adapters visually distinct so the lean-core story is obvious.
- Use the logo-derived organic nesting motif as the memorable visual element.
- Prefer dense, scannable documentation layouts over marketing-only composition.
- Verify desktop and mobile layouts so code samples, nav, and calls to action remain readable.
