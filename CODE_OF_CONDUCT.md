# Code of Conduct

Nidus contributors are expected to keep discussions technical, respectful, and
focused on improving the project.

## Expected Behavior

- Be direct without being hostile.
- Critique code, designs, and tradeoffs rather than people.
- Assume maintainers will ask for tests, docs, and smaller changes when needed.
- Keep issues and pull requests on topic.
- Respect private vulnerability handling described in `SECURITY.md`.

## Unacceptable Behavior

Harassment, personal attacks, discriminatory language, doxxing, spam, and
deliberate disruption are not acceptable in project spaces.

## Enforcement

Maintainers may edit or remove comments, close issues, block accounts, or take
other reasonable action to protect the project and its contributors.
