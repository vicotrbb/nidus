# Nidus Dashboard API Example

This example mounts Nidus Dashboard at `/nidus/dashboard` with bearer auth,
SQLite storage, metadata-only capture, route snapshots, event capture, job
capture, the runtime graph API, an SSE stream, and normal application routes.

The app is intentionally small but graph-shaped:

- `AppModule` imports `AccountsModule`, `BillingModule`, `NotificationsModule`, and `AuditModule`.
- `AccountsModule`, `BillingModule`, `NotificationsModule`, and `AuditModule` each declare providers and controllers.
- Feature modules export providers, and several modules import those exports.
- Controller routes are generated by Nidus macros; `/health` and `/activity/*` are manual utility routes.

Run it:

```bash
export NIDUS_DASHBOARD_TOKEN=dev-dashboard-token
export NIDUS_DASHBOARD_DATABASE_URL='sqlite://./target/nidus-dashboard-example.sqlite?mode=rwc'
cargo run -p nidus-example-dashboard-api
```

App route checks:

```bash
curl http://127.0.0.1:4310/health
curl http://127.0.0.1:4310/accounts/acme
curl -X POST http://127.0.0.1:4310/accounts
curl http://127.0.0.1:4310/billing/acme
curl -X POST http://127.0.0.1:4310/billing/invoices
curl -X POST http://127.0.0.1:4310/notifications/email
curl -X POST http://127.0.0.1:4310/notifications/sms
curl http://127.0.0.1:4310/audit/recent
```

Dashboard auth and API checks:

```bash
curl -i http://127.0.0.1:4310/nidus/dashboard/
curl -i -H 'Authorization: Bearer wrong' http://127.0.0.1:4310/nidus/dashboard/
curl -i -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/

curl -i http://127.0.0.1:4310/nidus/dashboard/api/overview
curl -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/api/overview
curl -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/api/graph
curl -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/api/routes
```

Seed realistic activity:

```bash
curl -X POST http://127.0.0.1:4310/activity/events/account-created
curl -X POST http://127.0.0.1:4310/activity/events/invoice-paid
curl -X POST http://127.0.0.1:4310/activity/events/notification-bounced

curl -X POST http://127.0.0.1:4310/activity/jobs/reconcile-billing
curl -X POST http://127.0.0.1:4310/activity/jobs/send-digest
curl -X POST http://127.0.0.1:4310/activity/jobs/audit-retention

curl -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/api/timeline
curl -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/api/graph
curl -N -H 'Authorization: Bearer dev-dashboard-token' http://127.0.0.1:4310/nidus/dashboard/stream
```

Expected security behavior:

- missing dashboard bearer token returns `401`
- invalid dashboard bearer token returns `401`
- valid dashboard bearer token returns the dashboard shell and APIs
- application routes remain normal app routes
- payload capture is off by default
